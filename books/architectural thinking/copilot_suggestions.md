GOAL 
    -  start with the logic of the program
    -  avoid wasting time on fluff
    -  
  read books below to learn how to achieve the goal

1. Clean Code by Robert C. Martin

Focus: Writing readable, maintainable code.
Why: Teaches principles that lead to better architecture through clean design.

2. Clean Architecture by Robert C. Martin

Focus: How to structure applications for flexibility and scalability.
Why: Introduces separation of concerns and layered architecture.

3. Design Patterns: Elements of Reusable Object-Oriented Software by Gamma et al.

Focus: Common solutions to recurring design problems.
Why: Helps you think in abstractions and reusable components.

4. The Pragmatic Programmer by Andrew Hunt & David Thomas

Focus: Practical tips for writing better software.
Why: Encourages thinking beyond just codingâ€”towards design and architecture.

5. Head First Design Patterns by Eric Freeman & Elisabeth Robson

Focus: Design patterns explained visually and interactively.
Why: Great for beginners to understand architectural patterns in an easy way.

6. Refactoring: Improving the Design of Existing Code by Martin Fowler

Focus: How to restructure code without changing behavior.
Why: Teaches how to evolve architecture from messy codebases.
